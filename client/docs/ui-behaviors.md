# UI Behaviors

このドキュメントは、auto-kanban クライアントの UI 動作仕様を定義します。

## チャットパネル (Chat Panel)

### スマートスクロール

チャットパネルでは、エージェント動作中にメッセージが追加された際のスクロール動作を以下のように制御します。

#### 動作仕様

| 状態 | 動作 |
|------|------|
| 一番下にスクロールしている | 新しいメッセージが追加されると自動的に一番下までスクロール（追従） |
| 一番下以外を見ている | スクロール位置を維持し、ユーザーが過去のメッセージを読める状態を保つ |

#### 実装詳細

- **追従判定**: スクロール位置が最下部から一定のしきい値（例: 50px）以内の場合、「一番下にいる」と判定
- **追従タイミング**: `entries.length` が変更されたときに追従判定を行い、条件を満たす場合のみスクロール
- **ユーザー操作の尊重**: ユーザーが上方向にスクロールした場合、自動追従を停止

#### 関連コンポーネント

- `client/src/components/chat/ChatContainer.tsx` - スマートスクロールの実装箇所

## タスクステータスワークフロー (Task Status Workflow)

タスクは以下のステータスを順に遷移します。自動遷移と手動遷移の両方をサポートします。

### ステータス一覧

| ステータス | 説明 |
|-----------|------|
| Todo | 未着手のタスク |
| In Progress | エージェントが作業中のタスク |
| In Review | エージェントが作業を完了し、レビュー待ちのタスク |
| Done | 完了したタスク |
| Cancelled | キャンセルされたタスク |

### ステータス遷移図

```
Todo ──(Start Agent)──> In Progress ──(Claude idle + no queue)──> In Review ──(PR merged)──> Done
  │                           │                                        │
  └───────────────────────────┴────────────────────────────────────────┴──> Cancelled (手動)
```

### 自動ステータス遷移

#### 1. Todo → In Progress
- **トリガー**: ユーザーが「Start Agent」ボタンをクリックしてエージェントを起動
- **条件**: エージェントの実行が開始された時点

#### 2. In Progress → In Review
- **トリガー**: Claude が idle 状態（ユーザー入力待ち）になった
- **条件**:
  - エージェントが `result` メッセージを送信し、idle 状態になった
  - キューされたメッセージがない
- **実装箇所**: `server/src/services/queue-processor.ts` または新規サービス

#### 3. In Review → Done
- **トリガー**: Pull Request がマージされた
- **条件**:
  - タスクに関連付けられたブランチの PR がマージされた
  - GitHub Webhook または定期的なポーリングで検出
- **実装箇所**: 新規 Webhook ハンドラまたはポーリングサービス

### 手動ステータス遷移

ユーザーは任意のタイミングでタスクのステータスを手動で変更できます:
- カンバンボード上でドラッグ＆ドロップ
- タスク詳細画面でステータスを選択

### メッセージ送信の動作

#### 即時送信 vs キュー

| エージェント状態 | 動作 |
|-----------------|------|
| 停止中 | メッセージを即時送信し、新しい実行を開始 |
| 実行中 + idle（入力待ち） | メッセージを即時送信（フォローアップ） |
| 実行中 + busy（処理中） | メッセージをキューに追加 |

#### キューされたメッセージの処理

1. エージェントの実行が完了（completed/failed/killed）
2. キューにメッセージがある場合、自動的に次の実行を開始
3. キューにメッセージがない場合、タスクを In Review に移動

### 関連コンポーネント

- `client/src/components/execution/FollowUpInput.tsx` - メッセージ送信 UI
- `client/src/hooks/useFollowUp.ts` - 送信/キューのロジック
- `server/src/services/queue-processor.ts` - キュー処理とステータス遷移
- `server/src/services/process-manager.ts` - プロセス完了時のコールバック
